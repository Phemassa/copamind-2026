Para Estruturação do RAG e Simulação (Totalmente Free & Sem Limites)Para alimentar o seu banco vetorial local (ChromaDB/FAISS) com os dados estruturados básicos (equipes, grupos, estádios e calendário completo das 104 partidas), use o modelo de hospedagem local:GitHub - OpenFootball WorldCup JSON: Um repositório open source clássico que disponibiliza os arquivos de dados brutos (worldcup.json) com todo o chaveamento e calendário da Copa de 2026 de forma estática. Perfeito para ler via Python localmente sem fazer requisições externas HTTP.GitHub - World Cup 2026 API (rezarahiminia): Você pode clonar este repositório e rodar o banco de dados MongoDB localmente via Docker Desktop. Ele já inclui scripts em Node.js prontos para ler arquivos CSV e injetar todas as informações da Copa. Além disso, conta com uma interface interativa via Swagger para você testar as rotas locais (/get/games, /get/teams) antes de criar suas ferramentas MCP.2. Para Resultados e Classificação em Tempo Real (Planos Gratuitos com Chave)Se o seu agente ou chat precisar buscar resultados reais e classificações de grupos atualizadas durante o torneio, você pode usar os planos gratuitos das APIs comerciais usando chaves de API individuais:API-Football (Plano Free): Fornece 100 requisições por dia de forma totalmente gratuita. É o suficiente se o seu servidor MCP possuir um sistema inteligente de cache local (guardando o JSON da resposta na memória ou em um arquivo local por alguns minutos para evitar chamadas repetidas desnecessárias).Football-Data.org (Plano Free): Inclui dados da Copa do Mundo em sua camada gratuita com um limite generoso de 10 chamadas por minuto. Muito boa para checar a tabela de classificação atualizada dos grupos.Zafronix APIs: Oferece um plano gratuito voltado especificamente para dados de Copas do Mundo (incluindo o histórico de edições passadas de 1930 até 2026), com um limite de até 250 requisições diárias. É uma excelente alternativa para criar o histórico de dados que treinará seus modelos preditivos locais.Recomendação de Arquitetura para o CopaMindDado que você está usando Python no VS Code, a melhor abordagem de custo zero é:Baixar o calendário estático das partidas em JSON direto do repositório do OpenFootball para servir de base para o simulador.Configurar o plano gratuito da API-Football apenas para as atualizações diárias de placar ao vivo e estatísticas de jogo.Gostaria que eu criasse um script de exemplo em Python que faz a leitura do arquivo JSON estático da Copa de 2026 para você começar a popular o banco de dados do seu RAG local?
