CopaMind 2026

Um experimento de benchmark com LLMs locais rodando em hardware proprio. Cada modelo percorre a mesma estrutura de tarefa e o mesmo contrato JSON. Os pacotes usam indices FIFA, analytics derivados e evidencias de jogadores disponiveis em cada execucao. O score e calculado fase a fase conforme os resultados oficiais chegam.

Fontes e tecnologia

Dados FIFA locais de selecoes, jogadores e partidas; analytics proprio em DuckDB; agente de chamadas para modelos locais via LM Studio; scoring do bolao por fase.

O portal publico e estatico. O Streamlit/Admin continua como console operacional para atualizar dados, rodar agentes e exportar snapshots.

Perfis que exigiram ajuste no ambiente

Durante a configuracao, 7 modelos exigiram um perfil explicito de Structured Output no LM Studio. Esta lista descreve setup; nao equivale a falhas finais do contrato.

  • zai-org / glm-4.7-flash
  • nvidia / nemotron-3-nano-4b
  • nvidia / nemotron-3-nano-omni
  • allenai / olmo-3-32b-think
  • qwen / qwen3.5-9b
  • qwen / qwen3.6-27b
  • qwen / qwen3.6-35b-a3b

Os demais 20 modelos da coorte usaram o perfil-base. No snapshot auditado, 3 modelos terminaram com pelo menos um slot sem aceite final.

Desclassificados

Modelos que nao conseguiram concluir o projeto de testes e foram removidos do ranking para nao contaminar os resultados.

  • mistralai / mistral-7b-instruct-v0.3 Channel Error no llama.cpp ao gerar saida estruturada. Sem Structured Output, nao retornava respostas validas para o contrato JSON do bolao. Modelo incompativel com o pipeline de palpites — removido da competicao.
